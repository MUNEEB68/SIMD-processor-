# SIMD-processor-

## Hardware Verification on DE1-SoC via JTAG Debug

This project contains a SIMD CPU core (`cputop.v`) and a debug observation tap (`cpu_debug_tap.v`) designed for hardware verification on a DE1-SoC FPGA board using Quartus Prime Lite 20.1. The current flow uses a **JTAG to Avalon Master Bridge** for debug reads (no HPS required).

### Architecture

- **cputop.v**: Main CPU with H/O/Q register banks and SIMD ALUs (add, multiply, shift, logic).
- **cpu_debug_tap.v**: Avalon-MM slave that exposes instruction address, opcode, data signals, and a sample counter.
- **platform_designer_setup.tcl**: Quartus Tcl script that auto-generates a minimal Qsys project skeleton.

### Setup Steps

#### 1. Generate the Qsys Project

Run the platform designer setup script:

```powershell
quartus_sh -t .\platform_designer_setup.tcl
```

This will:
- Create `simd_system.qsys` (empty Quartus 20.1 system file).
- Create `qsys_components/cpu_top/` and `qsys_components/cpu_debug_tap/` with your HDL files.
- Print next steps to the console.

#### 2. Open and Configure in Platform Designer

1. Open Quartus Prime and create a new project (or use an existing one targeting DE1-SoC / Cyclone V).
2. In Quartus, go to **Tools → Platform Designer** to launch the Qsys editor.
3. Open the generated `simd_system.qsys` file (File → Open).

#### 3. Add Components

**CPU Top Module:**
- In Platform Designer, go to File → New → Component from Files.
- Browse to `qsys_components/cpu_top/` and import `cputop.v`.
- Name it `cpu_top`, mark as "Elaborated HDL", set language to Verilog.
- Click Finish to create the component.
- Instantiate it in the system (double-click the system in the left panel, then drag `cpu_top` from the IP Catalog into the design).

**Debug Tap Module:**
- Similarly, go to File → New → Component from Files.
- Browse to `qsys_components/cpu_debug_tap/` and import `cpu_debug_tap.v`.
- Name it `cpu_debug_tap`.
- **Important:** Manually define the Avalon-MM Slave interface:
  - In the component editor, add a new interface of type "Avalon Memory Mapped Slave".
  - Set address width to 2 bits (registers at offsets 0x0, 0x4, 0x8, 0xC).
  - Set data width to 32 bits.
  - Map address/read/readdata signals to the Verilog ports.
- Click Finish and instantiate it in the system.

#### 4. Add JTAG Master (for debug)

1. In Platform Designer, go to **IP Catalog → Interface Protocols → JTAG**.
2. Add **JTAG to Avalon Master Bridge** to the system.
3. Connect its Avalon-MM master to `cpu_debug_tap_0.s0`.

#### 5. Connect Signals

1. Connect the debug tap as an Avalon-MM slave to the JTAG master:
   - In the System tab, drag from `master_0` to `cpu_debug_tap_0.s0`.
   - Assign a base address (e.g., 0x0000_0000).

2. Connect clock and reset:
   - Route `clk_0` (or HPS clock) to both `cpu_top` and `cpu_debug_tap`.
   - Route `reset_n` to both modules.

3. Wire the CPU signals to the debug tap:
   - In the Verilog instantiation or Platform Designer advanced mode, connect CPU outputs (instruction_in, instruction_address, data_address, data_out, data_R, data_W, done) to the debug tap inputs.

#### 6. Generate and Export

1. In Platform Designer, go to **Generate → Generate HDL**.
2. Choose output directory (typically your Quartus project folder).
3. Once generation is complete, Platform Designer will create:
   - `simd_system.qip` (Quartus IP file).
   - Generated Verilog/SystemVerilog HDL in a subdirectory.

4. Add the `.qip` file to your Quartus project (Project → Add Files).

#### 7. Build and Program

1. In Quartus, complete your top-level design:
   - Instantiate `simd_system` (from the generated HDL).
   - Connect CPU instruction/data memories and other I/O.
2. Compile the design (Processing → Start Compilation).
3. Program the board (Tools → Programmer).

### Debug Register Map

The `cpu_debug_tap` Avalon-MM slave provides these read-only registers:

| Address | Name | Contents |
|---------|------|----------|
| 0x0 | Control/Status | instruction_address (10 bits), opcode (6 bits), data_R, data_W, done |
| 0x4 | Data Signals | data_out (16 bits), data_address (10 bits) |
| 0x8 | Instruction | Full instruction word (18 bits) |
| 0xC | Sample Counter | Increments on each new CPU instruction (16 bits) |

### JTAG Debug (System Console)

Use **Tools → System Console** in Quartus to read the debug registers over JTAG.

Example (assumes base address `0x0000_0000`):

```tcl
# In System Console
set mm [get_service_paths master]
open_service master $mm
master_read_32 $mm 0x00000000 4
close_service master $mm
```

This returns the 4 debug registers in order: control/status, data signals, instruction, and sample counter.
